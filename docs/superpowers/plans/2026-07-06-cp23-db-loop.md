# CP 2.3 — Loopen mot databasen: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The game loop reads today's batch from Supabase and submit logs `game_sessions`/`decisions`; RLS proven (own rows only, `batch_cards` unreachable).

**Architecture:** psycopg3 pool (`backend/db.py`) behind a `Repo` protocol (`backend/repo.py`) injected as a FastAPI dependency — unit tests swap in `FakeRepo` built from the existing fixture; an idempotent `backend/seed.py` freezes the golden-fixture batch into the hosted DB.

**Tech Stack:** psycopg[binary,pool], FastAPI dependency_overrides, pytest (env-gated integration), PostgREST + GoTrue REST for the RLS proof.

## Global Constraints

- Backend deps via `uv add`; tests `uv run pytest backend/tests/ -q` from repo root.
- `SUPABASE_DB_URL` (session pooler, IPv4) already in `.env`; never committed.
- Reveal contract stays locked to 01 §7 — no shape changes.
- Sessions log `is_daily=false` in this CP (free replay; enforcement is CP 4.2).
- Commit per task; push at the end.

### Task 2: DB pool + Repo protocol with PgRepo

**Files:** Create `backend/db.py`, `backend/repo.py`; Test: `backend/tests/test_repo_fake.py` (FakeRepo contract only).

**Interfaces (produced):**
- `db.pool() -> psycopg_pool.ConnectionPool` (lazy, from `SUPABASE_DB_URL`).
- `repo.BatchMeta(batch_id, mode, status, decision_date, horizon_years, daily_date, R_m_cum, R_f_cum, intro: dict)`
- `repo.TruthRow(card_no, company_id, ticker, name, ret_cum, clue, event)`
- `class Repo(Protocol)`:
  - `get_live_daily(self) -> tuple[BatchMeta, list[Card]] | None` (Card = schemas.Card from public_payload)
  - `get_batch_meta(self, batch_id: int) -> BatchMeta | None`
  - `get_batch_truth(self, batch_id: int) -> list[TruthRow]`
  - `record_session(self, user_id, batch_id, mode, score, bonus, hit_rate, choices: list[ChoiceIn]) -> int` (transactional insert session + decisions; returns session_id)
- `PgRepo` implements it with SQL (joins `companies` for ticker); `get_repo()` FastAPI dependency returning a module-level `PgRepo`.

Steps: TDD the pure parts (BatchMeta/TruthRow construction + FakeRepo used by later tasks) → implement PgRepo (SQL exercised by Task 5 integration tests, not unit tests) → commit.
